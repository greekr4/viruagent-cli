# Doc Sync — 문서 자동 동기화

viruagent-cli에 새 기능/프로바이더/커맨드가 추가되었을 때, 관련 문서를 일괄 업데이트한다.

## 트리거 시점

- 새 프로바이더 추가 후
- 기존 프로바이더에 새 커맨드 추가 후
- 사용자가 "문서 업데이트", "README 갱신" 등 요청 시

## 업데이트 대상 및 위치

### 1. README.md (영문)

**위치**: 프로젝트 루트 `/README.md`

업데이트할 섹션:
- **Supported Platforms 테이블** — 새 플랫폼 행 추가
- **Usage 테이블** — 새 사용 예시 행 추가
- **Tech Stack 테이블** — 새 기술 행 추가 (해당 시)
- **Login 섹션** — 새 프로바이더 로그인 방법 추가 (해당 시)

### 2. README.ko.md (한글)

**위치**: 프로젝트 루트 `/README.ko.md`

README.md와 동일한 섹션을 한국어로 업데이트. 기존 번역 톤/스타일을 유지한다.

### 3. CLAUDE.md (프로젝트 지침)

**위치**: 프로젝트 루트 `/CLAUDE.md`

업데이트할 섹션:
- **프로젝트 구조 트리** — 새 파일/디렉토리 추가
- **프로바이더별 특징 테이블** — 새 프로바이더/기능 행 추가
- **환경변수 섹션** — 새 환경변수 추가 (해당 시)
- **Rate Limit 규칙** — 새 프로바이더의 rate limit 추가 (해당 시)

### 4. 스킬 파일 (skills/)

**위치**: `/skills/va-{provider}-{action}/SKILL.md`

기존 스킬 파일 형식을 따라 새 커맨드용 스킬 파일 생성:

```yaml
---
name: va-{provider}-{action}
description: "viruagent-cli {설명}..."
---
```

필수 섹션: 선행 조건, 실행 명령어, 파라미터 테이블, 에러 처리, 예시.

### 5. 라우터 스킬 (va-shared/SKILL.md)

**위치**: `/skills/va-shared/SKILL.md`

- **Platform Router 테이블** — 새 트리거 추가
- **단일 명령 스킬 테이블** — 새 스킬 행 추가

### 6. 프로바이더 개요 스킬

**위치**: `/skills/va-{provider}/SKILL.md`

- **명령 목록 테이블** — 새 커맨드 행 추가

## 실행 절차

1. **변경 파악** — `git diff` 또는 사용자 입력에서 추가된 커맨드/기능을 파악한다
2. **CLI 스펙 확인** — `npx viruagent-cli --spec`으로 현재 전체 커맨드 목록을 확인한다
3. **기존 문서 읽기** — 각 대상 파일을 Read로 읽어 현재 상태를 확인한다
4. **차이 계산** — CLI 스펙에는 있지만 문서에는 없는 커맨드를 찾는다
5. **업데이트 실행** — Edit 도구로 각 파일의 해당 섹션에 내용을 추가한다
6. **검증** — 업데이트 후 파일을 다시 읽어 포맷이 깨지지 않았는지 확인한다

## 규칙

- 기존 내용을 삭제하지 않는다 — 추가만 한다
- 기존 스타일/톤을 따른다 — 새 항목이 기존 항목과 이질적이면 안 된다
- 한글/영문 동시 업데이트 — README.md를 수정하면 README.ko.md도 반드시 수정한다
- 스킬 파일은 기존 스킬과 동일한 YAML frontmatter 형식을 사용한다
- 라우터 스킬 업데이트 시 기존 라우팅 테이블을 파괴하지 않는다
